# Code Reviewer Skill

## When to use

- User asks for a code review
- User wants feedback on their implementation
- User asks to check code quality

## Instructions

1. Read the file(s) the user wants reviewed
2. Analyze the code for:
   - Potential bugs or logic errors
   - Performance issues
   - Security concerns
   - Readability and maintainability
   - Adherence to project conventions
3. Provide feedback organized by severity:
   - **Critical**: bugs, security issues, data loss risks
   - **Warning**: performance issues, potential edge cases
   - **Suggestion**: readability, naming, style improvements
4. For each issue found, explain **why** it matters and suggest a fix
5. End with a brief summary of the overall code quality
